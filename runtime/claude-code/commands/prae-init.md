# /prae-init

> **Purpose**: Initialize a new PRAE research project
> **Arguments**: `[--name <project_name>]` (optional, defaults to the current directory name)
> **Preconditions**: You have run /prae-bootstrap to deploy the PRAE framework

This is the **project-state initialization entry point**.
It is not responsible for deploying PRAE into the project; that step is `/prae-bootstrap`. It is also not the model-context entry point; model context should first be established from `CLAUDE.md` / `AGENTS.md`.

## Execution Steps

### 1. Invoke the initialization tool

```bash
PROJECT_NAME="${1:-$(basename $(pwd))}"

python3 tools/init_project.py \
  --name "${PROJECT_NAME}" \
  --output-dir .
```

This tool will:
- Parse the infrastructure tracks and research tracks from `prae/PRAE_INIT.md`
- Generate `prae/track_registry.yaml`
- Generate the real `prae/phases/phase_00_infra/PHASE_BRIEF.md`
- Generate a `TRACK_LOG.md` and a directory skeleton for each infrastructure track

If the tool is not installed, do not manually copy templates to fake a `track_registry.yaml` or `PHASE_BRIEF.md`; restore the tool first, then run the same command.

### 2. Restore the tool (when the tool is unavailable)

```bash
if [ ! -f "tools/init_project.py" ]; then
  echo "Missing tools/init_project.py"
  echo "Re-run /prae-bootstrap first, or copy tools/init_project.py from the PRAE repository into the current project's tools/"
  exit 1
fi
```

### 3. Guide the user through filling in PRAE_INIT.md

Switch to the **Analyst** role and work through this with the user step by step:

1. **Problem Statement**:
   - "What core problem is this research project trying to solve?"
   - "What are the success criteria (quantifiable)?"
   - "Under what conditions should the entire project be terminated?"

2. **Infrastructure track identification**:
   - "Which capabilities will multiple research lines depend on? (for example, data ingestion, the backtest engine)"
   - Guide the user to name each infrastructure track (format: infra_{name}_v1)

3. **Research track identification**:
   - "What different algorithm directions or hypotheses do you want to validate?"
   - Guide the user to name each research track and write a hypothesis (format: research_{topic}_{variant})

### 4. Check the initialization results

```bash
python3 -c "
import yaml, os
r = yaml.safe_load(open('prae/track_registry.yaml'))
assert r['current_phase'] == 'phase_00_infra'
assert 'current_cycle' in r
assert os.path.exists('prae/phases/phase_00_infra/PHASE_BRIEF.md')
for t in r['tracks']:
    if t['type'] == 'infrastructure':
        log_path = f'prae/phases/phase_00_infra/tracks/{t[\"id\"]}/TRACK_LOG.md'
        assert os.path.exists(log_path), log_path
        content = open(log_path, encoding='utf-8').read()
        assert '{{' not in content, f'{t[\"id\"]} TRACK_LOG.md still contains template placeholders'
print('Initialization artifact check passed')
"
```

